# 🚀 TrafficReserve – 대규모 트래픽 대응 예약 시스템

TrafficReserve는 **많은 사용자가 동시에 예약을 시도해도 데이터가 꼬이지 않도록** 서버에서 요청을 제어하고,
예약 결과를 **실시간으로 확인할 수 있게 만든 예약 시스템**입니다.

선착순 이벤트나 인기 상품 예약 상황을 가정해, **백엔드 중심으로 설계된 프로젝트**입니다.

---

## 🛠 기술 스택 (Tech Stack)

| 구분       | 기술                        | 사용 목적                   |
| -------- | ------------------------- | ----------------------- |
| Backend  | Java, Spring Boot         | 예약 처리 및 트래픽 제어 핵심 로직 구현 |
| Frontend | React (Vite)              | 관리자용 실시간 예약 현황 대시보드     |
| Database | MariaDB (JPA / Hibernate) | 회원, 상품, 예약 데이터 저장       |
| 동시성 제어   | Redis, Redisson           | 동시에 들어오는 예약 요청 순서 제어    |
| 실시간 통신   | WebSocket (STOMP)         | 예약 결과 실시간 전달            |
| DevOps   | Docker, Docker Compose    | DB·Redis 실행 환경 구성       |

---

## 💡 핵심 구현 기능

### 1️⃣ 동시에 예약해도 재고가 정확히 관리되도록 처리

**상황**
여러 사용자가 동시에 하나 남은 상품을 예약할 경우,
재고 수량이 잘못 줄어들 가능성이 있음.

**구현 방식**

* Redis를 사용해 **한 번에 하나의 예약만 처리**되도록 제어
* 서버에서 예약 처리 순서를 직접 관리

**결과**

* 재고가 음수로 내려가는 문제 방지
* 동시에 많은 요청이 와도 데이터가 안정적으로 유지됨

---

### 2️⃣ 예약 처리와 알림 로직 분리

**구현 내용**

* 예약 처리 완료 후, 알림(로그) 로직을 별도로 실행
* 알림 로직은 예약 응답 이후 **비동기로 처리**

**효과**

* 사용자 응답 속도 개선
* 코드 구조 단순화 및 유지보수성 향상

---

### 3️⃣ 실시간 예약 현황 대시보드

**구현 내용**

* 예약 성공 / 실패 발생 시 서버에서 즉시 프론트엔드로 전달
* 관리자 화면에서 새로고침 없이 실시간으로 확인 가능

---

## ▶️ 실행 방법 (로컬)

```bash
# Docker 컨테이너 실행 (DB, Redis)
docker-compose up -d

# 백엔드 실행
./gradlew bootRun

# 프론트엔드 실행
npm install
npm run dev
```

## 📈 포트폴리오 요약

> 단순한 CRUD 프로젝트가 아니라,
> 동시에 많은 요청이 들어오는 상황을 가정해 서버에서 요청 순서를 제어하는 로직을 직접 구현했습니다.
> 또한 예약 결과를 실시간으로 확인할 수 있도록 실시간 대시보드를 구성했습니다.
